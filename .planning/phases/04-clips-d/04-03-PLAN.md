<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-entered URL/title/note/tag in ClipEditorView | Arbitrary text persisted to SwiftData; must be trimmed/validated at the save boundary, never crash |
| clip.url → openURL/Link (opening an arbitrary user URL) | The stored URL is opened in Safari; a malformed or non-web scheme must degrade gracefully, and the URL must be OPENED, never network-fetched (offline gate) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Denial of Service | malformed clip.url passed to `URL(string:)`/`Link(destination:)` | mitigate | Guard `URL(string:)` with a graceful fallback (disabled/no-op Open Link, no force-unwrap) so a bad URL never crashes the detail view |
| T-04-06 | Information Disclosure / offline-gate violation | any accidental fetch when "opening" a link or suggesting a title | mitigate | URLs are handed to `openURL`/`Link` (OS Safari handoff), never fetched; acceptance greps assert zero `URLSession`/`dataTask`/`.load`/`URLRequest` across all three files (SC1 / D-01) |
| T-04-07 | Tampering (input) | empty/whitespace title or url saved | mitigate | Save CTA disabled until `trimmedTitle` AND `trimmedURL` non-empty; fields trimmed at save (matches Rule's title gate) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs — DesignKit local path + SwiftUI + SF Symbols + Apple Link/openURL only; no new third-party dependency (UI-SPEC Registry Safety) |
</threat_model>
