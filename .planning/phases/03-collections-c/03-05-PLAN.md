<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported backup JSON → ExportImportService.importReplace | Untrusted external file; schemaVersion mismatch, malformed payload, or dangling FK ids (collectionID/domainID) must fail gracefully or wire to nil, never crash or silently corrupt |
| seed re-run → SeedDataService merge-add | Re-running restore must not duplicate the starter collection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering / Spoofing | import of a backup with a wrong schemaVersion | mitigate | `guard bundle.schemaVersion == schemaVersion (4) else throw ImportError.unsupportedSchema` — unchanged guard rejects mismatched payloads at the boundary with a user-facing error (CLAUDE.md §4) |
| T-03-10 | Denial (data integrity) | dangling collectionID/domainID on import; deleteAll ordering | mitigate | FK lookups via `flatMap(index[...])` resolve to nil (never force-unwrap) so orphan rows import cleanly; deleteAll deletes items → collections → domain to avoid orphaned cascade state |
| T-03-11 | Denial (duplication) | restoreMissingDefaults re-run duplicating the starter | mitigate | Merge-add guarded by an isSeeded/title/domain dedup key (D-14) — idempotent |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs — Foundation + SwiftData + XCTest only; no new third-party dependency |
</threat_model>
