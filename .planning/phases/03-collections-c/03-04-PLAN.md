<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted statusIndex → chip label lookup | A statusIndex may be out of range for the resolved StatusSet (e.g. imported/older item); indexing must not crash |
| user cost text field → Double parse | Free-text decimal input at the cost field is untrusted; a bad parse must fail gracefully to nil, not crash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Denial of Service (crash) | chip `statusSet.states[item.statusIndex]` and terminal lookup | mitigate | Use a bounds-safe subscript (`states[safe:]`) and clamp advance with `min(...)`; unresolved StatusSet falls back to empty label / terminal 0 rather than force-unwrap |
| T-03-08 | Tampering (input validation) | Cost decimal-pad TextField parse | mitigate | Parse via `Double(text)` → optional; invalid input stored as `nil`, never crashes; title guarded non-empty before save |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs — DesignKit local path + SF Symbols + Apple Link/openURL only; no new dependency |
</threat_model>
