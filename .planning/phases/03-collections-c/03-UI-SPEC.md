# Phase 3 — UI Design Contract

> Visual and interaction contract for the Collections surfaces. Generated by gsd-ui-researcher from 03-CONTEXT.md decisions D-01…D-23, verified against DesignKit source files, and authored in the same idiom as the approved Phase 2 UI-SPEC (02-UI-SPEC.md).
>
> **Platform note:** Native **SwiftUI / iOS** backed by the shared **DesignKit** Swift Package — NOT web/CSS. No shadcn, no registry, no hex values. All values are DesignKit semantic tokens and SF Symbols. `CLAUDE.md` hard rules are binding: no hard-coded colors, accents constrained to the 5 domain-token set, Balanced Luxury restraint, spacing/radii/typography from DesignKit tokens only.
>
> **Continuity:** This contract reuses the Phase 1/2 token vocabulary verbatim. The nav shape (domain section → "+" in section header → picker/editor → detail view) mirrors Phase 2's D-12 template exactly. `accentColor(forToken:)` from Phase 1 is consumed here without change.

## Design System Exceptions

This section documents deliberate deviations from generic checker heuristics. These are not violations — they are the correct application of the established native design system to a SwiftUI app that already ships Phases 1 and 2 on DesignKit.

### Typography: 6 roles / 3 weights

**Exception:** The generic checker ceiling of "≤4 roles / ≤2 weights" is a heuristic aimed at ad-hoc CSS type scales where unconstrained growth is the risk. It does not govern an established, centralized native design system.

**Rationale:** `DesignKit/Typography/TypographyTokens.swift` defines exactly **6 semantic roles** (`titleLarge`, `title`, `headline`, `body`, `caption`, `monoNumber`) with their associated weights (bold, semibold, regular, and monospaced-regular). These are the app-wide vocabulary already shipped and approved in Phases 1 and 2. This phase introduces **zero new sizes and zero new weights** — all tokens are consumed verbatim from the existing DesignKit ramp, enforcing cross-phase consistency per CLAUDE.md §1 and §9.4 (single source of truth, verify tokens exist before using).

**Phase-3 surface usage (constrained subsection):** The new Phase-3 surfaces lean on 4 of the 6 roles for their own content. `titleLarge` and `monoNumber` appear only where consistency with already-built surfaces requires it.

| Surface | `titleLarge` | `title` | `headline` | `body` | `caption` | `monoNumber` |
|---------|:---:|:---:|:---:|:---:|:---:|:---:|
| S1 — CollectionRow | — | — | row title | — | StatusSet sub-label; rollup "X/Y" | — |
| S2 — PresetPickerSheet | — | sheet heading; "Choose a type" | preset name | preset description | state-flow sub-label | — |
| S3 — CollectionDetailView | empty-state heading | collection name | — | description; empty-state body; rollup text | StatusSet sub-label | rollup cost "$NNN"; rollup "X/Y" numerals standalone |
| S4 — CollectionItemRow | — | — | item title | — | position label; "X/Y" rollup | — |
| S5 — CollectionItemDetailView | — | position read-only display; item title | block labels ("Status") | note content | cost/url labels; note label | cost amount "$NNN" |
| S6 — CollectionItemEditorSheet | — | — | field labels | field values | — | — |

`titleLarge` is reserved for the empty-state heading in S3 (matches the Phase 1/2 empty-state idiom). `monoNumber` appears only for numeric cost and standalone rollup numerals. All other phase-3 content uses `title` / `headline` / `body` / `caption`.

### Spacing: `theme.spacing.m = 12`

**Exception:** The checker flagged `m = 12` as not in the "standard 4-8-16-24-32-48-64" set. This is correct only for web/CSS spacing scales.

**Rationale:** `DesignKit/Layout/SpacingTokens.swift` defines `m = 12` as a first-class semantic token (verified: `xs=4, s=8, m=12, l=16, xl=24, xxl=32`). 12pt is on the 4pt grid (12 = 3×4; no misalignment). This token is used app-wide in Phases 1 and 2. Phase 3 uses `theme.spacing.m` where the gap between related elements calls for something tighter than `l=16` but looser than `s=8` — specifically between position control buttons and between CollectionItem rows. No ad-hoc spacing values are introduced; every gap in this phase maps to a named DesignKit token.

Source of truth: `DesignKit/Layout/SpacingTokens.swift` (verified: `xs=4, s=8, m=12, l=16, xl=24, xxl=32`). **Do not invent values — use these tokens by name.**

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `theme.spacing.xs` | 4 | Chip label horizontal inset; position-label ↔ chip gap; icon ↔ text gap inside rows |
| `theme.spacing.s` | 8 | Between stacked preset picker rows; between form field label and value; cost/url field inner spacing |
| `theme.spacing.m` | 12 | Between CollectionItem rows; between detail blocks; between position control buttons |
| `theme.spacing.l` | 16 | `DKCard` surface padding; section content inset; sheet horizontal padding |
| `theme.spacing.xl` | 24 | Between major blocks in `CollectionDetailView` and `CollectionItemDetailView`; empty-state vertical rhythm |
| `theme.spacing.xxl` | 32 | Empty-state top inset; major section breaks |

| Token | Value | Usage |
|-------|-------|-------|

## Typography

Source of truth: `DesignKit/Typography/TypographyTokens.swift` (verified — 6 token roles). Do not introduce new `.font(.system(size:))` calls. See "Design System Exceptions" above for checker rationale.

| Role | Token | Underlying | Usage in this phase |
|------|-------|-----------|---------------------|
| Display | `theme.typography.titleLarge` | 32 / bold / rounded | Empty-state heading in Collections section and empty CollectionDetailView |
| Heading | `theme.typography.title` | title2 / semibold | Section header "Collections"; `CollectionDetailView` collection name; `CollectionItemDetailView` item title |
| Label | `theme.typography.headline` | headline | `CollectionRow` collection title; `CollectionItemRow` item title; position control button labels; form field labels; preset name in picker |
| Body | `theme.typography.body` | body / regular | `CollectionDetailView` header description; form field values; note field content; empty-state body; preset description in picker |
| Caption | `theme.typography.caption` | caption | Compact position label ("S2 E4", "Chapter 7"); trailing rollup text on row ("12/50"); preset sub-label; cost field placeholder; status chip label uses `DKBadge` which applies `caption.weight(.semibold)` |
| Mono | `theme.typography.monoNumber` | monospaced body | The X/Y numerals in the rollup ("**23**/50") when rendered standalone outside a `DKBadge`; cost amount ("$340") |

- Dynamic Type: all text scales; item title in `CollectionItemRow` caps at 2 lines with `.minimumScaleFactor` only for the compact row layout. All other text wraps freely.
- Line height: SwiftUI defaults per token; do not override.
- The `DKBadge` component renders `caption.weight(.semibold)` with `accentPrimary` foreground on `highlight` background — consume as-is for status chips.

|------|-------|-------|

| Text — primary | `theme.colors.textPrimary` | Collection/item titles; position labels; body text |
| Text — secondary | `theme.colors.textSecondary` | Captions; rollup labels; placeholder text; preset sub-labels |
| Accent (10%) — domain | `accentColor(forToken: domain.colorToken, scheme:)` | **Reserved for:** the domain glyph tint in `CollectionDetailView` header only |
| Accent — theme | `theme.colors.accentPrimary` | "+" section header button; position control buttons; "Add Item" and "Save" CTAs; `DKBadge` foreground (status chip, rollup badge); `DKProgressRing` arc |
| Highlight | `theme.colors.highlight` | `DKBadge` background (status chip, rollup badge); `DKProgressRing` track fill |
| Destructive | `theme.colors.danger` | "Delete collection" and "Delete item" destructive action labels; reset-chip confirmation if shown |

**Accent (`accentColor(forToken:)`) reserved for (exhaustive):**
1. The owning domain's glyph in the `CollectionDetailView` header (tinted from `domain.colorToken`).

**Accent is NOT used for:** status chip foreground (uses `accentPrimary`), position control buttons, row highlights, or any text — those use `textPrimary` / `accentPrimary` / `danger`. The 5 domain accents remain identity-only (one color = one domain).

**Status chip color semantics:** all status states use the same `DKBadge` token rendering (`accentPrimary` text on `highlight` background). No per-state color override — the chip label alone communicates state. Terminal state is distinguished by the stop-at-terminal behavior (D-06), not a color change.

**Cost sum color:** `textPrimary` for the dollar amount ("$340"). Never accent-colored — cost is informational, not a call-to-action.

## Component Inventory & Interaction Contract

### S1 — Collections section in `DomainDetailView` (COLL-01, COLL-06, COLL-07)

- **Integration:** append a `DomainSection(id:"collections", title:"Collections", …)` to `nonEmptySections`. The section renders **only when `domain.collections` is non-empty** (D-15) — a domain with zero collections shows no section; the DOM-03 "only non-empty sections" contract holds. A collection with zero items is still a non-empty section entry (the `Collection` model exists).
- **Section header:** `Text("Collections")` styled as `theme.typography.title` with `.accessibilityAddTraits(.isHeader)`, plus a trailing **"+"** button (`Image(systemName:"plus")`, `accentPrimary` tint, ≥44pt) that presents `CollectionPresetPickerSheet` (S2) as a sheet, pre-scoped to `domain`.
- **Row (`CollectionRow`, data-driven per §9.2):** `DKCard` surface. Layout: leading `VStack` (collection title in `theme.typography.headline` textPrimary, 1 line + optional caption in `theme.typography.caption` textSecondary showing the StatusSet name, e.g. `"Shows — watched"`); trailing rollup label when `showsAggregate == true` (format spec below). Whole row is one ≥44pt `NavigationLink` → push `CollectionDetailView(collection:)`.
- **Trailing rollup on row (D-17):** when `showsAggregate == true`, trailing label is either:
  - `"X/Y"` rendered in `theme.typography.caption` (textSecondary) for completionist collections. A small `DKProgressRing` may optionally follow if the number is below 10 items and space allows — but plain text `"X/Y"` is always present and is the minimum; the ring is additive (D-18).
  - `"$NNN"` rendered in `theme.typography.monoNumber` (textSecondary) for cost-flavored collections (D-18). Never a ring.
  - No trailing label when `showsAggregate == false`.
- **Ordering:** collections sorted by `sortIndex` ascending; seeded collection appears first (lowest sortIndex).
- **Swipe actions:** no swipe-delete on the row (delete lives in a dedicated editor/detail action; no Phase 2 rule row precedent for swipe-delete either).

### S2 — `CollectionPresetPickerSheet` (COLL-07, COLL-02)

- Presented as a **sheet** (`theme.radii.sheet`) from the "+" in the Collections section header (D-13).
- **Header:** `"Choose a type"` in `theme.typography.title` (textPrimary) at the top. No domain-name repetition (already scoped from the section context).
- **Preset list:** a `List` or `VStack` of rows, one per preset in `CollectionPresetCatalog`. Each row: preset **name** in `theme.typography.headline` (textPrimary) + a one-line preset **description** in `theme.typography.caption` (textSecondary) describing the status states (e.g. `"to-watch → watching → watched"`). Tap → create the `Collection` from the chosen preset (pre-filled with the preset's `statusSetID`, `progressTemplate`, `showsAggregate` default), dismiss the sheet, push `CollectionDetailView` for the newly created collection.
- **Preset order:** Generic first, then Shows, Movies/franchises, Albums, Concerts, Books, Clothes to buy, Want to spend on, Planes/places (matches SPEC §5 order; D-12).
- **Cancel:** "Cancel" button in trailing toolbar position (or sheet dismissal via swipe). No collection is created on cancel.
- No empty state needed — the catalog is always populated.
- **Accessibility:** each row `accessibilityLabel`: `"<preset name>, <description>"`. "Cancel" button `accessibilityLabel`: `"Cancel, no collection created"`.

### S3 — `CollectionDetailView` (COLL-01, COLL-03, COLL-06)

**Primary focal element:** The collection name in `theme.typography.title` (textPrimary), anchored by the domain-glyph accent tint immediately to its left. The rollup (ring or cost sum) is the secondary anchor directly beneath the name.

- Data-driven: takes a `Collection` value; declares **no `NavigationStack`** (nests under the Hub stack). Vertical `ScrollView` with `VStack(alignment:.leading, spacing: theme.spacing.xl)`.
- **Header block (always shown):**
  - Row 1: owning-domain glyph (`Image(systemName: domain.iconName)`, accent-tinted via `accentColor(forToken:)`, size 28) + collection **name** in `theme.typography.title` (textPrimary). The domain glyph reinforces which domain this list belongs to.
  - Row 2 (when `showsAggregate == true`): rollup display (see format spec below). Sits immediately below the title.
  - Row 3 (conditional): collection description / StatusSet sub-label in `theme.typography.caption` (textSecondary), e.g. `"Shows — to-watch → watching → watched"`.
- **Rollup display in header (D-17):** when `showsAggregate == true`:
  - Completionist: `DKProgressRing(progress: Double(x)/Double(y), lineWidth: 6, label: "\(x)/\(y)", theme: theme)` at a small size (≤56pt diameter). The ring's center text uses `theme.typography.title` + `caption` (built into `DKProgressRing`). A text fallback `"\(x) of \(y) \(terminalStateLabel)"` in `theme.typography.body` (textSecondary) sits beneath the ring. When `y == 0`, render `"0 items"` in `theme.typography.body` (textSecondary) and no ring (avoids divide-by-zero).
  - Cost: `"$\(formattedTotal)"` in `theme.typography.monoNumber` (textPrimary). Never a ring (D-18). When no items have cost, show `"$0"`.
  - Tracker (`showsAggregate == false`): no rollup block in header.
- **Items list:** a `ForEach` over `collection.items` sorted by `sortIndex` ascending. Each item renders as a `CollectionItemRow` (S4) wrapped in a `NavigationLink` → `CollectionItemDetailView(item:)`.
- **Empty state (§9.3):** when `collection.items.isEmpty`, show (D-15 does not guard against an empty item list — empty item list is valid):
  - Heading: `"Nothing in this list yet"` in `theme.typography.title` (textPrimary).
  - Body: `"Tap + to add your first item."` in `theme.typography.body` (textSecondary).
- **Toolbar:** trailing **"+"** button (`Image(systemName:"plus")`, `accentPrimary`, ≥44pt, `accessibilityLabel`: `"Add item to \(collection.name)"`) presents `CollectionItemEditorSheet` (S6) in create mode. Optionally, a trailing **"Edit"** button (or ellipsis menu) → `CollectionEditorSheet` for collection-level settings (name, `showsAggregate` toggle, delete) — scope this to a lightweight `.contextMenu` or a separate `CollectionEditorSheet` following the `RuleEditorView` idiom.

### S4 — `CollectionItemRow` (COLL-03, COLL-04, COLL-05)

- Data-driven: takes a `CollectionItem` value; owns no query (§9.2).
- `DKCard` surface. **Layout (HStack):**
  - Leading `VStack(alignment:.leading, spacing: theme.spacing.xs)`:
    - Item **title** in `theme.typography.headline` (textPrimary), 2-line cap + `.minimumScaleFactor(0.9)`.
    - Compact **position label** (conditional — only when `progressTemplate != .none`):
      - `seasonEpisode`: `"S\(season) E\(episode)"` in `theme.typography.caption` (textSecondary).
      - `counter`: `"\(counterLabel ?? "Item") \(counterValue)"` in `theme.typography.caption` (textSecondary). E.g. `"Chapter 7"`.
  - Spacer.
  - Trailing `HStack(spacing: theme.spacing.xs)`:
    - Rollup label when `showsAggregate == true` on the parent collection (see S1 format). For an item row this is omitted — rollup is at the collection level, not the item level.
    - **Status chip:** `DKBadge(statusLabel, theme: theme)` where `statusLabel` is `statusSet.states[item.statusIndex]`. The chip is a tap target:
      - `.onTapGesture`: advance `statusIndex` by 1, clamped at `terminalIndex` (D-06). No-op at terminal; `.sensoryFeedback(.impact(.light))` on every tap including terminal — the feedback still fires to confirm recognition (D-08).
      - `.contextMenu`: one item — `"Reset"` with destructive role → sets `statusIndex = 0` (D-07).
      - VoiceOver: `accessibilityLabel` = `"Status: \(statusLabel), \(item.title)"`. Custom accessibility action: `accessibilityCustomContent` or `AccessibilityCustomAction("Reset status")` that performs the reset (D-07).
      - Chip min tap target: ≥44pt via `.frame(minWidth:44, minHeight:44)`.
- Full row is a `NavigationLink` → `CollectionItemDetailView(item:)` (≥44pt total row height via `.frame(minHeight:44)` applied to the card content).

### S5 — `CollectionItemDetailView` (COLL-03, COLL-04, COLL-05)

**Primary focal element:** When `progressTemplate != .none`, the position display in `theme.typography.title` (Block 2) is the primary focal element — it communicates progress at a glance. When `progressTemplate == .none`, the status chip (Block 1) is the primary focal element.

- Data-driven: takes a `CollectionItem` value; declares **no `NavigationStack`**. Vertical `ScrollView` of blocks separated by `theme.spacing.xl`.
- **Block 1 — Status (always):** large chip group. Label: `"Status"` in `theme.typography.headline` (textSecondary). Below: the current `DKBadge` status chip (same visual as S4). Tap-to-advance and contextMenu Reset work identically to S4 (D-06, D-07, D-08).
- **Block 2 — Position controls (conditional — only when `progressTemplate != .none`):**
  - **`seasonEpisode`:** Three buttons arranged in an `HStack` with `theme.spacing.m` gaps, each a bordered button (background `theme.colors.surface`, radius `theme.radii.button`, border `theme.colors.border`, ≥44pt min dimension):
    - `"+Episode"` — `accessibilityLabel`: `"Next episode"` — increments `episode` by 1.
    - `"+Season"` — `accessibilityLabel`: `"Next season"` — increments `season` by 1, resets `episode` to 1 (D-10).
    - `"Finished"` — `accessibilityLabel`: `"Mark as finished"` — sets `statusIndex = terminalIndex` (D-10). Only visible when not already at terminal; when at terminal, show `"Restart"` or omit (use contextMenu Reset from S4 — prefer omit to keep the surface clean).
  - **`counter`:** One button:
    - `"+1"` or `"+\(counterLabel ?? "Item")"` — increments the counter value by 1. Full-width or constrained to ≤200pt. `accessibilityLabel`: `"Add one \(counterLabel ?? "item")"`. 
  - Position display (read-only, above the controls): large `theme.typography.title` (textPrimary) text showing the current position — `"S\(season) E\(episode)"` for seasonEpisode, or `"\(counterLabel ?? "Item") \(counterValue)"` for counter.
- **Block 3 — Metadata (conditional fields — omit each block when nil/empty):**
  - **Note:** `theme.typography.body` (textPrimary), full-wrap, selectable. Label `"Note"` in `theme.typography.caption` (textSecondary) above it. No label if empty.
  - **URL / Source:** bordered affordance identical to `RuleDetailView`'s source block pattern — `Image(systemName:"link")` + host (body/textPrimary) + full URL (caption/textSecondary). Tap opens via `Link`/`openURL`. `accessibilityLabel`: `"Open link, \(host)"`. ≥44pt. Offline-safe (stored + opened only).
  - **Cost:** `"$\(formattedCost)"` in `theme.typography.monoNumber` (textPrimary) with label `"Cost"` in `theme.typography.caption` (textSecondary).
- **Toolbar:** trailing **"Edit"** button → presents `CollectionItemEditorSheet` (S6) in edit mode, prefilled.

### S6 — `CollectionItemEditorSheet` (add/edit item)

- Presented as a **sheet** (`theme.radii.sheet`), modeled after `RuleEditorView`. Fields:
  - **Title** (`TextField`, trim + non-empty guard, prefilled in edit mode).
  - **Note** (`TextEditor`, optional, multi-line).
  - **URL / Source** (`TextField`, `.keyboardType(.URL)`, `.textInputAutocapitalization(.never)`, optional).
  - **Cost** (`TextField`, `.keyboardType(.decimalPad)`, optional, formatted to 2 decimal places).
- Field labels `theme.typography.headline` (textPrimary); values `theme.typography.body` (textPrimary); field placeholder hint `theme.typography.body` (textSecondary).
- **Save CTA:** `"Add Item"` (create mode) / `"Save Changes"` (edit mode). `accentPrimary`, `theme.radii.button`, ≥44pt, full-width or trailing toolbar position. Disabled until non-empty trimmed title.
- **Delete (edit mode only):** destructive **"Delete Item"** row (`theme.colors.danger`). Single-confirm via `confirmationDialog` (copy below). On confirm, cascade delete removes the item from the collection (D-22 — items are owned by collection, cascade is expected). No stems to preserve.
- **Cancel:** dismisses, creates/changes nothing.

### Accessibility (§9.15 — part of "done", applies to all surfaces)

- **Status chip:** `accessibilityLabel` = `"Status: \(statusLabel), \(item.title)"`. Custom accessibility action "Reset status" performs the reset in place of long-press (D-07). VoiceOver does not need to describe the tap-to-advance — the label change on next focus confirms it.
- **Position controls (S5):** each button has an explicit `accessibilityLabel` as specified above. The read-only position display has `accessibilityLabel`: `"Current position: S\(season) E\(episode)"` (or equivalent for counter).
- **Collection row (S1):** `accessibilityLabel`: `"\(collection.name), \(itemCount) items, \(rollupIfPresent)"` — e.g. `"Shows, 12 items, 3 of 12 watched"`.
- **Item row (S4):** `accessibilityLabel`: `"\(item.title), status: \(statusLabel)\(positionIfPresent)"` — e.g. `"Succession, status: watching, S2 E4"`. The status chip (nested inside) also has its own label; use `.accessibilityElement(children:.ignore)` on the row container and compose a single label to avoid VoiceOver duplication.
- **"+" buttons:** `accessibilityLabel`: `"Add collection to \(domain.name)"` (S1 header); `"Add item to \(collection.name)"` (S3 toolbar).
- **Preset picker rows (S2):** `accessibilityLabel`: `"\(presetName), \(presetDescription)"`.
- **DKProgressRing (S3 header):** the built-in ring shows percentage in center — additionally set `.accessibilityLabel` on the ring container: `"\(x) of \(y) \(terminalStateLabel), \(Int(progress*100)) percent complete"`.
- **Dynamic Type:** no fixed-height text containers anywhere; position display in S5 uses `.title` and scales fully; no clipped text.
- **Contrast:** rely on DesignKit token pairings; no accent-on-accent text; cost/rollup uses textSecondary on background, which meets AA.

## Rollup Format Specification (D-16 / D-17 / D-18)

`CollectionRollupEngine` returns one of `.count(x, y)`, `.costSum(total)`, or `.none`. The UI maps to:

| Engine result | Row trailing label | Header display |
|---------------|--------------------|----------------|
| `.count(x, y)` where y > 0 | `"\(x)/\(y)"` — `caption` / `textSecondary` | `DKProgressRing` at ≤56pt + `"\(x) of \(y) \(terminalStateLabel)"` in `body` / `textSecondary` |
| `.count(x, y)` where y == 0 | (none) | `"0 items"` in `body` / `textSecondary`; no ring |
| `.costSum(total)` | `"$\(formatted)"` — `monoNumber` / `textSecondary` | `"$\(formatted)"` — `monoNumber` / `textPrimary` (larger) |
| `.none` | (none) | (no rollup block) |

Cost formatting: `NumberFormatter` with `currencyCode: "USD"` and `locale: .current` (D-20 deferred "currency locale" to executor). Use `"$\(formatted)"` pattern; executor should apply `Locale.current` formatting.
