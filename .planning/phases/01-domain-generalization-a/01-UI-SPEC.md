# Phase 1 — UI Design Contract

> Visual and interaction contract for the new Hub surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform note:** This is a native **SwiftUI / iOS** app backed by the shared **DesignKit** Swift Package — NOT web/CSS. There is no shadcn, no registry, no hex. All values below are expressed as **DesignKit semantic tokens** and **SF Symbols**. "Hard rules" from `CLAUDE.md` are binding: no hard-coded colors, accents constrained to exactly 5 tokens (forest, navy, maroon/oxblood, walnut, stone), Balanced Luxury restraint, spacing/radii/typography come from DesignKit tokens only.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI; no shadcn/registry) |
| Preset | not applicable — DesignKit `ThemePreset` is user-selected at runtime; this contract is preset-agnostic |
| Component library | DesignKit (`../DesignKit`, local path) — `DKCard`, `DKBadge`, `DKSectionHeader`, `DKButton`, `DKProgressRing` |
| Icon library | **SF Symbols only** (Apple SDK). Custom-domain icons restricted to a curated `[String]` set of ~30 (D-16). No third-party icon dependency. |
| Font | DesignKit `TypographyTokens` (system / `.rounded` for titleLarge). Dynamic Type respected (§9.15) |
| Color source | DesignKit `theme.colors.*` semantic tokens, accessed via `themeManager.theme(for: colorScheme)`. **NEW** app-level `accentColor(forToken:)` resolver (research Pattern 4) maps the 5 stored tokens → `Color`; lives in app `Utilities/`, NOT DesignKit (§9.14). |

**Hard constraint (binding on every surface below):** No `Color(hex:)`, no `Color(red:…)`, no literal radii/spacing/opacity. Colors come only from `theme.colors.*` or `accentColor(forToken:)`. Spacing only from `theme.spacing.*`; radii only from `theme.radii.*`; type only from `theme.typography.*`.

## Spacing Scale

Source of truth: `DesignKit/Layout/SpacingTokens.swift` (verified). **Do not invent values — use these tokens by name.**

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `theme.spacing.xs` | 4 | Icon-to-label gap inside a tile; chip inner vertical padding |
| `theme.spacing.s` | 8 | Selected-ring inset; swatch-row item gap; badge padding |
| `theme.spacing.m` | 12 | Inter-tile grid spacing; icon-grid cell gap |
| `theme.spacing.l` | 16 | Card padding (DKCard default); section horizontal padding; segmented-control horizontal padding |
| `theme.spacing.xl` | 24 | Between DomainDetailView sections; empty-state vertical rhythm |
| `theme.spacing.xxl` | 32 | Empty-state top inset / major breaks |

Radii — source `DesignKit/Layout/RadiusTokens.swift`:

| Token | Value | Usage |
|-------|-------|-------|
| `theme.radii.card` | 16 | Hub tiles, DKCard surfaces |
| `theme.radii.button` | 14 | Buttons in creation form |
| `theme.radii.chip` | 12 | Swatch corner (if rounded-rect) / pills |
| `theme.radii.sheet` | 22 | Icon-picker & creation sheets |

**Exceptions:** Tile and swatch tap targets must be **≥ 44×44 pt** (HIG / §9.15 accessibility), even though no spacing token equals 44 — set via `.frame(minWidth:44,minHeight:44)`, not a magic padding. This is the only allowed non-token dimension and it is an accessibility floor, not a style choice.

## Typography

Source of truth: `DesignKit/Typography/TypographyTokens.swift` (verified). Use these **6 token roles** by name — do not introduce new `.font(.system(size:))` calls. Weights are baked into the tokens (titleLarge=bold, title=semibold, body/caption=regular), satisfying the "restrained weight set" intent.

| Role | Token | Underlying | Usage in this phase |
|------|-------|-----------|---------------------|
| Display | `theme.typography.titleLarge` | 32 / bold / rounded | Hub nav title, empty-state heading |
| Heading | `theme.typography.title` | title2 / semibold | DomainDetailView section headers (via `DKSectionHeader`), domain name in detail header |
| Label | `theme.typography.headline` | headline | Tile domain name; focus-picker row labels; creation form field labels |
| Body | `theme.typography.body` | body / regular | Empty-state body copy, descriptive text, picker subtitles |
| Caption | `theme.typography.caption` | caption | "new domains" hint text, swatch/icon a11y captions, "N domains focused" counts |
| Mono | `theme.typography.monoNumber` | monospaced body | Not used this phase (reserved for counters in later phases) |

- Dynamic Type: all text must scale; tiles use a 2-line cap with `.minimumScaleFactor` only where truncation would break the grid. No fixed-height text containers.
- Line height: governed by SwiftUI defaults per token; do not override.

## Color

DesignKit applies a **60/30/10 split structurally** via semantic tokens — this contract maps each role to the existing token, it does not pick new colors. Verified against `DesignKit/Theme/Tokens.swift` (`ThemeColors`).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `theme.colors.background` | Screen background for Hub, DomainDetailView, focus picker, sheets (Balanced Luxury: warm cream light / charcoal dark) |
| Secondary (30%) | `theme.colors.surface` / `theme.colors.surfaceElevated` | Hub tiles (DKCard), detail-section cards, sheet surfaces, focus-picker rows |
| Border | `theme.colors.border` | Tile/card stroke (1pt), swatch unselected outline |
| Text | `theme.colors.textPrimary` / `textSecondary` / `textTertiary` | Primary labels / supporting copy / disabled-empty hints |
| Accent (10%) | `accentColor(forToken:)` → one of forest/navy/maroon/walnut/stone | **Reserved list below — never "all interactive elements"** |
| Destructive | `theme.colors.danger` | Unfocus/remove-tile affordance text, delete-domain confirmation (the only destructive surface this phase) |
| Selected fill | `theme.colors.fillSelected` | Selected state of an icon-grid cell / swatch backing |

**Accent reserved for (exhaustive):**
1. The domain's own glyph + tile tint on a Hub tile (`DomainTile`), resolved from that domain's stored `colorToken`.
2. The per-domain icon color in `DomainDetailView`'s header.
3. The selected-ring around the chosen swatch in `DomainColorSwatchRow` (D-17).
4. The selected-cell highlight in `DomainIconPicker` (D-16).
5. The 5 color choices presented in the swatch row themselves (each renders its own token).

**Accent is NOT used for:** body text, section chrome, the segmented control, tab bar items, nav titles, or generic buttons — those use `textPrimary`/`surface`/`accentPrimary` defaults so the 5 domain accents stay meaningful (one color = one domain identity).

**Resolver contract (NEW — research Pattern 4 / DOM-03 / D-17):**
- `accentColor(forToken token: String, theme: Theme) -> Color` accepts exactly `"forest" | "navy" | "maroon"(alias "oxblood") | "walnut" | "stone"`.
- Unknown / legacy token → fall back to `theme.colors.accentPrimary` (safe default; never crash, never off-palette).
- Source the 5 colors from the matching `ThemePreset` accents via `PresetCatalog`/`PresetTheme` (confirm exact public accessor at build time — research A2). The resolver is the **single** place a token string becomes a `Color`; both the Hub tile and the swatch row consume it (no duplication).
- `colorToken` is **valid-by-construction**: the only writer is the closed 5-swatch row, so no validation UI is needed.

## Component Inventory & Interaction Contract

### S1 — Hub tile grid (`HubView` + `DomainTile`) — DOM-03
- Layout: `LazyVGrid` of adaptive tiles, inter-item spacing `theme.spacing.m`, screen padding `theme.spacing.l`. Parent (`HubView`) owns the `@Query` filtered on `isFocused == true`; `DomainTile` is **data-driven** props only (§9.2) — name, iconName, colorToken — and runs no query.
- Tile visual: `DKCard` surface (`theme.radii.card`), centered SF Symbol tinted via `accentColor(forToken:)`, domain name below in `theme.typography.headline` (textPrimary), 2-line cap.
- Interaction: whole tile is one ≥44pt tap target → push `DomainDetailView(domain:)` onto the Hub `NavigationStack`. No long-press / no swipe actions this phase.
- **Empty state (mandatory, §9.3 / POL-02 down-payment):** when zero focused domains — heading + body + a button that deep-links into the focus picker. Copy below.

### S2 — `DomainDetailView` — DOM-03
- Renders **only non-empty sections** (Phase 1 has no offshoot content yet, so in practice this is the header + an empty state). Use `DKSectionHeader` per section.
- Header: domain glyph (accent-tinted) + name in `theme.typography.title`.
- **Empty state (mandatory):** since Rules/Collections/Clips/Ideas arrive in later phases, an unfocused-content domain shows the empty state. Copy below.

### S3 — `DomainFocusPicker` — DOM-04 + D-10
- A `List`/`Form` of all domains (catalog), each row: glyph + name + a focus toggle. Toggling flips `isFocused` only — **never deletes content** (unfocus hides the Hub tile, data persists).
- Row label uses `theme.typography.headline`; toggle uses the system `Toggle` (inherits tint). Destructive delete (custom domains only) via swipe → confirmation (see Copywriting).
- **"New domains available" hint (D-10) — prescribed treatment:** a single subtle **`DKBadge`** reading `New` placed inline next to newly merge-added (`isFocused == false`, `isSeeded == true`, seedVersion-tagged) catalog rows, plus one-line caption header at top of the picker. Chosen badge-over-banner: less intrusive, matches Balanced Luxury restraint, and is per-row discoverable rather than a dismissible interruption. Caption uses `theme.typography.caption` / `textSecondary`.

### S4 — Custom-domain creation (`DomainIconPicker` + `DomainColorSwatchRow`) — DOM-05 / D-16 / D-17
- Presented as a sheet (`theme.radii.sheet`). Three inputs: **Name** (TextField, trim + length-guard like existing `CategoryManagerView`), **Icon** (curated grid), **Color** (5-swatch row).
- `DomainIconPicker`: `LazyVGrid` of ~30 hand-picked SF Symbols (static `[String]`, Claude's discretion to finalize — lifestyle-relevant, consistent with seeded glyphs). Cells ≥44pt; selected cell backed by `theme.colors.fillSelected` with an accent ring. Data-driven: parent passes selection binding.
- `DomainColorSwatchRow`: horizontal row of the **5 accent tokens** rendered via `accentColor(forToken:)`; the selected swatch carries an accent **ring** (selected-state, D-17). No color wheel, no hex field — `colorToken` valid-by-construction.
- Primary CTA: see Copywriting. On save: insert + `try modelContext.save()` (match existing pattern); new domain appears in the catalog (DOM-05).

### S5 — Progress Charts ⇄ Calendar segmented control — DOM-06 / D-13 / D-14
- `Picker("", selection:).pickerStyle(.segmented)` with two segments **Charts** and **Calendar**, horizontal padding `theme.spacing.l`, at the top of `ProgressDashboardView`. Progress owns the single `NavigationStack`; Calendar view's inner `NavigationStack` + `navigationTitle("Calendar")` are stripped (D-14). Default selection: **Charts**. No accent on the control — system tint only.

### Accessibility (§9.15 — part of "done", applies to all surfaces)
- VoiceOver labels: every tile → `"<Domain name>, domain"`; every swatch → its color name (e.g. `"Forest"`) + selected state; every icon-grid cell → symbol's human name + selected state; focus toggle → `"Focus <name>"` with on/off value.
- Dynamic Type: no clipped text; tiles reflow rather than truncate hard.
- Contrast: relies on DesignKit token pairings (textPrimary on surface/background) — do not place accent-on-accent text.
- Tap targets ≥ 44pt on tiles, swatches, and icon cells.
