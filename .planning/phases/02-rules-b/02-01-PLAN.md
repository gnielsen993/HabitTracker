<objective>
Add the `Rule` `@Model` and the bidirectional Rule<->Habit stem link as pure schema-expansion under plan-less inferred migration (Docs/SCHEMA_MIGRATION_PLAYBOOK.md). Rule gains `title`, `body`, optional `sourceURL`, `createdAt`, additive-defaulted `isArchived: Bool = false`, a `domain: Domain?` relationship (mirroring `Habit.category`), and a `stemmedHabits` inverse. `Habit` gains `originRule: Rule?` with `@Relationship(deleteRule: .nullify, inverse: \Rule.stemmedHabits)` — so deleting a rule nulls the pointer and never cascades to habits (RULE-05). Register `Rule` in the container type list, bump Export/Import to `schemaVersion = 3` with a `RuleDTO` array plus a `originRuleID` on `HabitDTO`, and ship unit tests for the nullify behavior + round-trip. Gate the whole change behind the manual upgrade test.

Purpose: This is the schema foundation the Rules UI (02-02) and the shared habit-create sheet (02-03) file into. The stem link is modeled as a SwiftData relationship (D-01), NOT a raw UUID — it delivers RULE-05's "nulled, never cascade" for free and gives RULE-04's count/backref directly. All new fields are optional or defaulted so inferred migration stays lightweight (playbook Step 1).
Output: Rule.swift, Habit with originRule, container registration, schemaVersion-3 Export/Import, passing RuleModelTests + ExportImportTests, and a passed (or pivoted) upgrade test.
</objective>
