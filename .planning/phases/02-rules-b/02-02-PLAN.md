<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-entered sourceURL -> openURL | A user-supplied URL string is opened by the system browser |
| rule delete action -> stemmed habits | A destructive UI action whose blast radius must stay on the rule |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0202-01 | Tampering/Availability | Delete Rule button reachable in editor | mitigate | confirmationDialog soft-confirm with explicit "habits kept" copy (RULE-05); actual deletion relies on 02-01's `.nullify` so habits cannot cascade |
| T-0202-02 | Information disclosure | sourceURL opened via openURL | accept | Offline constraint honored — the app performs NO fetch/preview; it only hands a user's own stored URL to the system browser on explicit tap. No auto-navigation, no metadata scraping |
| T-0202-03 | Availability | oversized rule body / malformed URL string | mitigate | Body is plain text (no execution); URL stored as String and only opened via openURL which no-ops on an invalid URL — no crash path |
| T-0202-SC | Supply chain | external package installs | accept | N/A — zero new dependencies; DesignKit tokens + SF Symbols + Apple Link only |
</threat_model>
