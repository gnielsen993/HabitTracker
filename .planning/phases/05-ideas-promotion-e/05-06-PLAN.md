<objective>
Teach the three existing target editors to accept an idea prefill and consume the idea on Save (via `PromoteService`), and add the `PromoteToCollectionPicker` list step — so Promote is "menu → existing prefilled editor → Save" with maximal reuse (D-06/D-07, IDEA-04/IDEA-05). No bespoke PromoteSheet; every field the editors already own is reused.

Purpose: IDEA-04's asymmetry — Rule gets title/body/sourceURL, Habit hands off to the shared sheet (RULE-02's load-bearing reuse), Collection item gets title + a user-picked list. On any successful promote-Save the idea is archived-with-forward-link and leaves the active list; the result has no backref. IDEA-05's missing-context prompts: an unfiled idea's Rule promote requires a domain before Save; promote-to-collection prompts for the target list. Rule and CollectionItem editors self-consume on save; the Habit editor stays chrome-clean and exposes an additive `onSaved` completion so its promote-caller (IdeaRow, 05-07) performs the consume.
Output: three editor edits (smallest diffs) + `PromoteToCollectionPicker.swift` (new).
</objective>
