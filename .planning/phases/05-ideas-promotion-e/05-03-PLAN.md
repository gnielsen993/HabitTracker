<objective>
Build `PromoteService` — the one small, pure, testable core that consumes an idea on promote: archive it, set a scalar forward-link to what it became, and enforce the "unfiled idea needs a domain" / "collection promote needs a list" preconditions (D-07, D-12, §9.5). All three target editors (05-06) call into this single helper on Save rather than each re-implementing the consume logic.

Purpose: IDEA-04's asymmetry (`DEC-promote-is-consume`) is the connective tissue — an idea that's promoted is archived-with-forward-link, leaves the active list, and the result carries no backref. Keeping that logic in one place makes it §9.5-testable and prevents the three editors from drifting. Designed pure-friendly (mutates passed-in models, no `ModelContext.save()` in the core) so its tests run in the engine tier that actually executes on the iOS 26 sim (§9.7), unlike @Model-persistence suites.
Output: `PromoteService.swift` + `PromoteServiceTests.swift` in the SAME commit (§9.5).
</objective>
